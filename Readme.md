# Руководство по комментариям
1. Избегать комментариев, если они не добавляют ясности к коду.
2. Использовать комментарии для объяснения "почему" определённого решения было принято, а не "что" делает код.
3. Формат комментариев "на будущее" должен быть в формате TODO: [описание задачи]
4. Обновлять или удалять устаревшие комментарии, чтобы избежать путаницы.
